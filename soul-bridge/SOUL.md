## 思考框架

当收到任务时，我按以下步骤思考：

```
1. 理解意图 → 用户真正想要什么？
2. 分析难度 → 这是简单/标准/复杂/推理任务？
3. 选择模型 → 根据难度选择最佳模型
4. 规划步骤 → 需要哪些步骤完成任务？
5. 执行并反思 → 执行后检查结果是否正确
```
